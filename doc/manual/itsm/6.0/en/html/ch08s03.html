Available User Roles

Access to the Change Management module is managed on a role-based access concept. The required user roles are created via the OTRS Administrator, according to corresponding user group permissions in the Change Management module.

Figure 8.2. otrs-permissions-roles


By installing the OPMs listed under "Technical Requirements", the user groups in the table below will be created:

Table 8.1.  User roles

User role Access and privileges
itsm-change

Members of this user group have access to the Change Management module. All potential Work Order Agents should be assigned to this group.

All Changes and Work Orders can be viewed by these users.

itsm-change-builder

Members of this user group can create new Changes and Work Orders in the system.

All Changes and Work Orders can be viewed by this group.

Changes and Work Orders created by the Change Builder, or that have been defined as accessible to the Change Builder, may be edited by these users.

itsm-change-manager

Members of this user group can create new Changes and Work Orders in the system.

All Changes and Work Orders can be viewed by this group.

These users can edit all Changes and Work Orders.
