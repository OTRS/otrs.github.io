Diagram of Change Management in OTRS::ITSM

The Change Management implementation in OTRS::ITSM defines a Change as an alteration of the existing IT landscape, such as the installation of a new mail server.

As Changes typically consist of several sub-tasks, OTRS::ITSM allows any number of sub-tasks to be defined per change. These are known as Work Orders.
